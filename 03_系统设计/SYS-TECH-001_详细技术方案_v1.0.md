## 1. 产品概述

### 1.1 产品定位

EL-1000 是一款 1000W 可编程直流电子负载，面向电源研发、电池测试、老化筛选等场景。对标 ITECH 8500 系列，采用 ITECH 风格外观，定位中端市场。

### 1.2 关键规格 (SRD v1.3)

| 参数 | 值 | 备注 |
|------|-----|------|
| 额定功率 | 1000 W | 峰值 1200 W (<30s) |
| 额定电压 | 150 V | |
| 额定电流 | 60 A | |
| 最小工作电压 | 1.5 V @ 60A | |
| 工作模式 | CC / CV / CR / CW | CR/CW 固件规划中 |
| 分辨率 | 0.01 A / 0.01 V | HRPWM 15.2 bit |
| 精度 | ±0.1% FS (V) / ±0.2% FS (A) | 校准后 |
| 纹波 | <10 mVrms | |
| 通信 | USB / Wi-Fi / UART / CAN | |
| 显示 | 4.3" IPS 彩屏 | ESP32 LVGL |
| 外形 | W310×D370×H120mm | ITECH 风格 |
| 整机 BOM | ¥1,354 | 小批量 50-200 台 |
| 目标零售 | ¥3,999–5,999 | |

### 1.3 工作模式说明

| 模式 | 控制策略 | 说明 |
|------|---------|------|
| **CC** (Constant Current) | 电流 PID → Duty | 直接控制负载电流，最常用模式 |
| **CV** (Constant Voltage) | 电压 PID → 电流 PID → Duty | 级联控制，稳压在输入端 |
| **CR** (Constant Resistance) | 电阻 PID → 电流 PID → Duty | 等效固定电阻负载（规划中） |
| **CW** (Constant Power) | 功率 PID → 电流 PID → Duty | 恒定功率吸收（规划中） |

## 3. 功率拓扑

### 3.1 MOSFET 阵列

**选型**: M90N20 (200V/90A TO-247)

| 参数 | 值 | 说明 |
|------|-----|------|
| Vds(max) | 200 V | 150V 降额 75% |
| Id(max) | 90 A | 单管 |
| Rds(on) | 22 mΩ | Vgs=10V |
| gm | 20 S (typ@8A) | |
| Pd(max) | 310 W | Tc=25°C |
| 数量 | 12 管 | 原型 10 管可用 |
| 总 Id | 1080 A | 远超 60A 需求 |
| 并联均流 | 源极 0.05Ω 电阻 | 12 × 0.05Ω 并联 = 0.004167Ω |
| 绝缘 | AlN 氮化铝垫片 | 热阻 0.4°C/W |

**并联结构**:
```
               ┌─── M90N20_1 ── 0.05Ω R_sense_1 ──  GND
               │
   Gate_drive ──┼─── M90N20_2 ── 0.05Ω R_sense_2 ──  GND
    (D882/B772) │         ...    (共 12 路)
               └─── M90N20_12 ─ 0.05Ω R_sense_12 ── GND
               
               DUT+ ────────────────────────────── MOSFET Drain
               所有 Source 在 GND 侧通过 RS6332 差分采样
```

**均流分析**:
- 源极电阻 0.05Ω 提供负反馈：电流大的管 Vgs 自动降低
- 12 管并联等效 R_sense = 0.004167Ω
- 满载 60A 时源极电阻压降: 60 × 0.004167 = 0.25V
- ADC 采样电压 (RS6332 增益 10): 0.25 × 10 = 2.5V (在 3.3V ADC 范围内)

### 3.2 栅极驱动

**方案**: D882(NPN) + B772(PNP) 推挽射极跟随器

| 参数 | 值 | 说明 |
|------|-----|------|
| 拓扑 | 互补推挽 | 快速充放电 Cgs |
| 驱动电压 | ~3.0 V (0.95×3.3V) | RC 滤波后 |
| 峰值电流 | ~1 A | D882/B772 能力 |
| 关断速度 | <1 µs | N7002N 下拉加速 |
| 数量 | 2 组推挽 (驱动 12 管) | 每组驱动 6 管 |

**驱动路径**:
```
C2000 HRPWM → RC LPF (10k+1µF, fc=15.9Hz) → Vdac (0~3.135V)
    → LM358 缓冲 → D882/B772 推挽 → 12× M90N20 Gate
                                          │
                                    N7002N 下拉电阻 10kΩ
```

**HRPWM→DAC 转换**:
```
PWM: 25 kHz, 占空比 D = 0~0.95
Vdac = D × 3.3V = 0 ~ 3.135V (经 RC 滤波)
等效分辨率: log2(25kHz / 15.9Hz × 0.95) ≈ 15.2 bit

替代方案: GP8101 (PWM→模拟, ¥2.5, SOP-8)
         可免去 RC 滤波，直接出 0~5V 模拟电压
```

### 3.3 电流检测

| 元件 | 规格 | 说明 |
|------|------|------|
| 检流电阻 | MCSF2512DR050Q9 (0.05Ω, 3W, 2512) | 12 颗并联 |
| 差分放大器 | RS6332 (替代 LM358, Vos 0.8mV) | 增益 10 倍 |
| 增益 | 0.04167 V/A (等效) | 60A → 2.5V ADC 输入 |

**精度预算**:
```
检流电阻精度: ±1% (MCSF2512)
RS6332 Vos: 0.8mV (max)
60A 时信号: 2.5V
Vos 误差: 0.8mV / 2.5V = 0.032%
校准后可达成 ±0.2% FS
```

### 3.4 电压检测

| 元件 | 规格 | 说明 |
|------|------|------|
| 分压比 | 1:13.2 | 150V → 11.36V → 3.3V ADC |
| 分压电阻 | 高精度 ±0.1% | 温度系数 25ppm |
| ADC 输入 | 0~3.3V | C2000 内置 ADC (12bit) |
| 等效分辨率 | 150V / 4096 ≈ 36.6mV/LSB | 软件过采样提升 |

## 6. 安全架构

### 6.1 5 道防护体系

| 层级 | 防护 | 时间 | 器件 | 触发条件 |
|------|------|------|------|---------|
| **L1** | 硬件 CMPSS | <1 µs | CMPSS1/2 (C2000) | V>150V 或 I>110A |
| **L2** | 硬件 PWM Trip | <100 ns | TZ 强制 Hi-Z | CMPSS 触发 |
| **L3** | 栅极下拉 | <15 µs | N7002N | GPIO 故障信号 |
| **L4** | 继电器断开 | 5-10 ms | 功率继电器 × 2 | 软件保护触发 |
| **L5** | 熔丝终级 | — | 125A 快断熔丝 | 前 4 级均失效 |

### 6.2 保护阈值

| 保护项 | 软件阈值 | 硬件阈值 | 动作 | 恢复 |
|--------|---------|---------|------|------|
| OVP | 150V | 155V (CMPSS1) | PWM Trip + 继电器 | 手动清除 |
| OCP | 110A | 115A (CMPSS2) | PWM Trip + 继电器 | 手动清除 |
| OPP | 1050W | — | 功率折返 | 自动 |
| OTP 降额 | 75°C | — | 功率线性降额 | 自动 |
| OTP 关断 | 90°C | — | PWM 关断 | 手动清除 |
| UVLO (PSU) | 10.8V | — | 系统安全关断 | 自动 |
| 通信超时 | 500ms | — | 保持 + 警告 | 自动 |

### 6.3 故障处理流程

```
故障触发
    │
    ├── L1/L2 (硬件): 直接 Hi-Z, <1µs
    │   └── 记录故障标志到 EEPROM
    │
    ├── L3 (软件保护): PID 任务检测
    │   ├── OVP/OCP: 立即关断 PWM
    │   ├── OTP: 75°C 折返 → 90°C 关断
    │   └── OPP: 功率限制
    │
    └── 恢复: 需手动清除 (CMD_CLEAR_FAULT)
          └── 急停也可物理复位
```

### 7.1 散热方案

| MOSFET × 12 | 铝挤压散热器 | 350×150×50mm |

| 散热风扇 | PWM 双 120mm | 12V, 可调速 |
| 热管理策略 | PID 控温 | 目标 65°C 以下 |

### 7.2 热阻链

```
Tj (MOSFET 结温)
    │  θjc = 0.3°C/W (M90N20)
Tcase
    │  θcs = 0.4°C/W (AlN 垫片)
Theatsink
    │  θsa = 0.06°C/W (350×150×50mm 铝挤 + 2.5m/s 风冷)
Tambient (40°C)
```

**1000W 满载温升**:
```
单管功耗: P_total / 12 = 83.3W (假设均匀)
Tj = 40 + 83.3 × (0.3 + 0.4 + 0.06) = 40 + 83.3 × 0.76
Tj = 40 + 63.3 = 103.3°C ✅ (M90N20 Tjmax=175°C, 降额 59%)
```

### 7.3 风扇策略

| 温度区间 | 风扇转速 | 说明 |
|---------|---------|------|
| <45°C | 0% (停转) | 静音模式 |
| 45-55°C | 20-40% | 低速散热 |
| 55-70°C | 40-80% | 中速 |
| 70-85°C | 80-100% | 全速 |
| >85°C | 100% + 功率折返 | 过热保护 |

### 8.2 UART 帧格式 (C2000 ↔ ESP32)

```
Sync0  Sync1  Length  Cmd    Payload    CRC16
 0xAA   0x55    1B     1B    0-255B     2B
```

| 命令 | 编码 | 方向 | 说明 |
|------|------|------|------|
| CMD_SET_MODE | 0x01 | ESP→C2000 | CC/CV/CR/CW |
| CMD_SET_CURRENT | 0x02 | ESP→C2000 | 设置电流 (mA) |
| CMD_SET_VOLTAGE | 0x03 | ESP→C2000 | 设置电压 (mV) |
| CMD_SET_OUTPUT | 0x04 | ESP→C2000 | ON/OFF |
| CMD_GET_TELEM | 0x10 | ESP→C2000 | 请求遥测 |
| CMD_TELEM_DATA | 0x11 | C2000→ESP | 遥测数据 |
| CMD_CLEAR_FAULT | 0x20 | ESP→C2000 | 清除故障 |
| CMD_SET_CAL | 0x30 | ESP→C2000 | 校准命令 |
| CMD_GET_STATUS | 0x40 | ESP→C2000 | 状态查询 |

**遥测数据包 (CMD_TELEM_DATA, 28 字节)**:
```
Offset  Size  Field           Unit
  0      4    V_load          mV
  4      4    I_load          mA
  8      4    P_load          mW
  12     4    V_fan_pwm       %
  16     4    Temp_1          °C×100
  20     4    Temp_2          °C×100
  24     4    Status flags    bitmask
```

### 8.3 SCPI 命令集 (USB/Wi-Fi 用户接口)

| 命令 | 参数 | 示例 |
|------|------|------|
| `:INPut:STATe` | ON/OFF | `:INPut:STATe ON` |
| `:SOURce:FUNCtion` | CC/CV/CR/CW | `:SOURce:FUNC CC` |
| `:SOURce:CURRent` | 0-60.00 | `:SOURce:CURR 5.00` |

## 9. 控制算法

### 9.1 算法架构

**级联控制**:
```
Setpoint ──► Mode Select ──► Outer Loop PID ──I_demand──► Inner Current PID ──Duty──► HRPWM
                                                           ↑
                                                    Current Feedback (860Hz)
```

### 9.2 PID 实现

**并联式 PI (离散)**:
```
error[n]       = setpoint - measurement[n]
p_term[n]      = Kp × error[n]
i_term[n]      = i_term[n-1] + Ki × error[n] × dt
output[n]      = p_term + i_term
output[n]      = clamp(output, min, max)
```

**抗饱和**:
```
if output >= max AND error > 0:  i_term[n] = i_term[n-1]  (冻结)
if output <= min AND error < 0:  i_term[n] = i_term[n-1]  (冻结)
else:                            i_term[n] = i_term[n-1] + Ki × error × dt
i_term[n] = clamp(i_term[n], ±max_integral)
```

### 9.3 PI 参数 (初始值)

| 环路 | Kp | Ki (离散) | Kd | 输出范围 | 说明 |
|------|----|----------|-----|---------|------|
| 电流环 (内环) | 0.5 | 10.0 | 0.001 | 0~0.95 duty | Ts=1.163ms |
| 电压环 (外环) | 0.2 | 2.0 | 0.0 | 0~110A I_demand | 带宽<<电流环 |

**仿真验证结论** (pid_sim.py):
```
电流环:
- 相位裕度: 109° (稳定)
- 闭环带宽: 112 Hz
- max_integral 需 ≥ 0.88 (30A 稳态 duty ≈ 0.88)
- 最终取 max_integral = 0.95
```

### 9.4 采样时序

```
PID 周期 T = 1/860 ≈ 1.163 ms

┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ ADC     │  │ PID     │  │ HRPWM   │  │ Sleep   │
│ @20µs   │─►│ @10µs   │─►│ @5µs    │─►│ ~1.13ms │
└─────────┘  └─────────┘  └─────────┘  └─────────┘
                ↑
           CMPSS 同步检查 (保护)
```

### 9.5 软启动

```
每次模式切换或使能输出:
1. 目标值 = setpoint
2. 100 步斜坡 (116 ms)
3. 每步: current_target = target × (step / 100)
4. PID 输出限幅随斜坡递增
```

## 13. 关键接口定义

### 13.1 电气接口

**输入 (DUT 侧)**:
```
DUT+ ────── 红色 4mm 镀金香蕉座 (30A)
DUT- ────── 黑色 4mm 镀金香蕉座 (30A)
Sense+ ──── 红色 2mm 香蕉座 (Kelvin 远端采样)
Sense- ──── 黑色 2mm 香蕉座 (Kelvin 远端采样)
```

**电源接口**:
```
AC 输入 ──── IEC C14 插座 (含保险丝座 10A/250V)
```

**通信接口**:
```
USB ──────── Type-C (USB 2.0 CDC, 虚拟串口)
LAN ──────── RJ45 (预留, Wi-Fi 优先)
CAN ──────── DB9 (隔离 CAN 2.0B)
```

### 13.2 控制接口

```
急停 ─────── 红色蘑菇头按钮 (常闭触点, 串联继电器线圈)
风扇 ─────── 4P PWM 端子 (12V/PWM/Tacho/GND)
```

### 13.3 固件接口 (C2000 ↔ ESP32)

参见第 8 节 UART 协议。注意:
- C2000 是 **从设备** (Wait-for-command)
- ESP32 是 **主设备** (发起所有事务)
- C2000 最长响应时间 **1ms**
- ESP32 轮询间隔 **10ms** (默认)
- WebSocket 推送间隔 **100ms** (10 Hz telemetry)
